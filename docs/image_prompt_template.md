# 图生图提示词模板

## 1. 汇总结论

基于 `图生图问卷.xlsx` 的有效反馈，团队当前图生图提示词的高频有效元素主要有：

- 明确要求 `产品本身不能变`
- 允许 `更换背景 / 场景 / 人物 / 构图 / 摆放`
- 强调 `1:1 电商主图`
- 强调 `避免明显拼贴感、分栏感和低质模板感`
- 强调 `像一张专业拍摄的广告图`
- 强调 `图片丰富、层次感、小物件、光影细节`
- 强调 `不要额外 logo / 水印 / 无关文字`
- 部分场景需要 `保留产品原有包装文字`
- 部分功效类产品需要 `少量强引导文案` 辅助转化

因此，提示词应聚焦五件事：

1. 锁定产品主体不变
2. 放开背景、场景、构图和氛围
3. 明确电商主图审美目标
4. 明确禁止项，减少模型误改产品
5. 对功效类产品，在合规前提下加入少量高转化文案、营销表达和数字化信息

## 2. 任务模式判定

第一版不需要让运营手动选择任务模式，直接根据 `主图数量` 和 `副图数量` 自动判定：

- `主图数量 > 0` 且 `副图数量 = 0`
  - `任务模式 = 仅主图`
- `主图数量 = 0` 且 `副图数量 > 0`
  - `任务模式 = 仅副图`
- `主图数量 > 0` 且 `副图数量 > 0`
  - `任务模式 = 整套图`
- `主图数量 = 0` 且 `副图数量 = 0`
  - 视为无效任务，直接拦截

补充规则：

- `仅副图` 模式必须提供已有主图或风格参考图，否则副图风格容易漂移。
- `整套图` 模式默认先生成主图，再审核主图，之后再规划和生成副图。
- `仅主图` 模式只走主图生成与审核链路。
- 第一版不做 `指定副图` 模式，避免入口字段变复杂。

## 3. 套图工作流分流

### `整套图`

适用条件：

- `主图数量 > 0`
- `副图数量 > 0`

建议流程：

1. 读取 `产品白底图`、`使用图`、卖点、风格要求、合规要求
2. OC 判断 `prompt_mode`
3. OC 生成主图 prompt
4. Seedream 生成主图
5. 主图审核
6. OC 输出 `sub_01 ~ sub_08` 的职责规划
7. OC 为各副图槽位生成 prompt
8. Seedream 生成副图
9. 副图审核
10. 交付整套图片

### `仅主图`

适用条件：

- `主图数量 > 0`
- `副图数量 = 0`

建议流程：

1. 读取任务输入
2. OC 判断 `prompt_mode`
3. OC 生成主图 prompt
4. Seedream 生成主图
5. 主图审核
6. 交付主图

### `仅副图`

适用条件：

- `主图数量 = 0`
- `副图数量 > 0`

建议流程：

1. 读取 `产品白底图`
2. 读取 `已有主图` 或 `风格参考图`
3. OC 判断 `prompt_mode`
4. OC 先输出副图职责规划
5. OC 为副图槽位生成 prompt
6. Seedream 生成副图
7. 副图审核
8. 交付副图

约束：

- `仅副图` 模式下，没有主图风格基准就不允许直接开跑。
- 副图风格应服从现有主图，而不是独立另起一套视觉语言。

## 4. 副图职责规划

第一版保留固定槽位：

- `sub_01`
- `sub_02`
- `sub_03`
- `sub_04`
- `sub_05`
- `sub_06`
- `sub_07`
- `sub_08`

但不把每个槽位的内容写死。
OC 需要先看任务输入，再动态决定每个槽位的职责。

### 候选职责池

- 核心卖点图
- 第二卖点图
- 第三卖点图
- 使用场景图
- 补充使用场景图
- 细节特写图
- 补充细节图
- 材质/成分图
- 功能结构图
- 使用步骤图
- 效果体验图
- 尺寸比例图
- 搭配展示图
- 氛围收尾图
- 组合展示图

### 规划原则

- 材质不重要时，不强行生成材质图。
- 使用场景更重要时，可以分配多张使用场景图。
- 卖点较多时，可以分配多张卖点图。
- 饰品类通常优先分配佩戴图、细节图、搭配图，不优先分配功效图。
- 功效类产品可以优先分配卖点图、使用场景图、效果图、规格图。

### 建议输出结构

```json
{
  "sub_image_plan": [
    { "slot": "sub_01", "role": "核心卖点图" },
    { "slot": "sub_02", "role": "第二卖点图" },
    { "slot": "sub_03", "role": "使用场景图" },
    { "slot": "sub_04", "role": "细节特写图" },
    { "slot": "sub_05", "role": "补充使用场景图" },
    { "slot": "sub_06", "role": "补充卖点图" },
    { "slot": "sub_07", "role": "搭配展示图" },
    { "slot": "sub_08", "role": "氛围收尾图" }
  ]
}
```

先有 `sub_image_plan`，再按槽位生成 prompt。不要一开始就让模型无约束地产出 8 张随机副图。

## 5. Prompt Mode 设计

为了避免每次都自由发挥，第一版工作流建议把图生图分成三种固定模式：

### `visual_only`
适用于：

- 饰品类
- 配饰类
- 外观型商品
- 情绪价值型商品
- 以质感、构图、氛围感为核心的产品

规则：

- 不增加额外文案
- 不做营销语强化
- 只突出产品质感、摆放、光影、场景和商业感

### `benefit_copy`
适用于：

- 洗发水
- 清洁剂
- 个护清洁
- 有明确痛点、效果或规格表达需求的产品

规则：

- 允许一句大字主文案
- 允许少量营销表达
- 允许数字化信息强化
- 但必须人工可审核、不可夸张失真

### `strict_packaging_only`
适用于：

- 产品包装本身已经带有关键文案
- 当前任务只允许保留原包装信息
- 对新增字样特别敏感的商品

规则：

- 只保留产品原包装上的文字
- 不允许在画面中新增任何文案
- 重点放在画面优化和产品突出

## 6. OC 的职责边界

OC 在这里的职责不是自由写整套提示词，而是：

1. 判断当前任务适合哪种 `prompt_mode`
2. 判断当前任务属于 `仅主图 / 仅副图 / 整套图`
3. 输出结构化变量
4. 在重做时根据驳回原因决定是否切换模板或重写局部 prompt
5. 在整套图模式下，先输出副图职责规划，再生成副图 prompt

建议 OC 输出：

```json
{
  "task_mode": "full_set",
  "prompt_mode": "visual_only",
  "reason": "该产品属于饰品类，主图重点应为质感、摆放和高级感，不适合添加额外文案。",
  "variables": {
    "产品卖点": [],
    "风格要求": "",
    "使用图场景摘要": "",
    "营销表达": [],
    "数字信息": []
  }
}
```

人工可以覆盖 `prompt_mode` 判断结果。

## 7. 推荐主模板

适用于：

- Seedream 图生图
- 输入包含 `产品白底图`
- 可选输入 `使用图`

```text
请基于参考图进行图生图创作，生成适合电商平台使用的高质量商品主图。

核心要求：
1. 产品主体必须与参考图保持一致，不要改变产品的外形、结构、颜色、材质、包装样式和品牌识别信息。
2. 如果产品包装本身带有文字或 logo，仅保留产品包装原有内容，不要在画面其他区域新增任何文字、logo、水印、贴纸、角标或平台元素。
3. 画面输出为 1:1 比例，可以是常规商品主图、佩戴展示图、细节展示图或场景图，但不要出现明显分栏排版或低质模板感。
4. 允许优化背景、场景、构图、光影、道具、人物动作或使用氛围，但所有变化都必须服务于突出产品本身，不要喧宾夺主。
5. 整体视觉要符合跨境电商主图审美：干净、精致、吸引点击、主体突出、信息传达明确、具有真实商业质感。
6. 基于任务提交字段中的地区， 选择对应的语言； 所有图片中的文案必须符合当地语言。

产品表现要求：
- 产品必须清晰完整，主体突出，边缘准确，不变形，不融化，不错位，不出现多余结构。
- 产品在画面中应占据视觉中心，保持真实比例。
- 可以根据产品特点增加适量场景元素、小道具、层次感、光影细节或氛围元素，但不能削弱产品主体。

场景与风格要求：
- 背景风格请根据产品定位进行优化，画面要自然、高级、统一。
- 如果提供了使用图，请参考使用图中的场景氛围、构图逻辑或使用状态，但仍以产品白底图作为产品真实性基准。


- 如果未提供使用图，请围绕产品卖点构建简洁、高级、适合电商点击的展示场景。

卖点表达要求：
- 不要直接在画面中写卖点文字。
- 通过构图、材质、光线、人物状态、使用场景和道具来间接传达产品卖点。

禁止项：
- 不要改变产品
- 不要新增无关产品
- 不要出现错误手部、错误肢体、畸形人物
- 不要出现模糊主体
- 不要出现低质塑料感
- 不要出现截图感、模板感
- 不要出现平台 logo、水印、边框、价格标签、促销贴纸

输出目标：
请生成一张看起来像真实商业摄影拍摄完成、适合 TikTok Shop / 跨境电商主图使用的精致图片。
```

## 8. `visual_only` 模板

适用于：

- 饰品类
- 展示型产品
- 不需要额外文案的主图

```text
请基于参考图生成新的电商商品图，保持产品完全一致，只优化展示方式与视觉质感。

产品锁定要求：
- 产品外观、颜色、材质、包装、标签位置、品牌信息、结构细节必须与参考图一致。
- 不允许擅自改变产品形态，不允许重新设计产品，不允许生成相似但不同的产品。

画面目标：
- 生成 1:1 的高质量电商主图。
- 画面应为自然、完整、适合电商展示的商品图，可以是佩戴图、细节图或场景图，但不要出现明显低质模板感。
- 主体突出、背景干净、氛围高级、点击感强。

规则：
- 不要在画面中新增文案。
- 不要加入营销语。
- 不要加入说明性文字。
- 仅通过背景、场景、光影、摆放、道具和构图来提升转化感。

可变化内容：
- 背景颜色
- 场景环境
- 道具搭配
- 光影层次
- 拍摄角度
- 产品摆放方式

变化原则：
- 一切变化都必须围绕产品质感服务。
- 画面可以更丰富，但不能让背景抢走主体。
- 可以加入少量辅助道具、小物件、反光、阴影、水珠、漂浮元素等细节，增强质感和层次感。

风格参考：
- {风格要求}

场景参考：
- {使用图场景摘要}

禁止项：
- 不要额外加文字
- 不要额外加 logo
- 不要额外加水印
- 不要拼贴
- 不要多宫格
- 不要错误人物结构
- 不要让产品比例失真
- 不要让产品看起来廉价或失焦

请输出一张真实、精致、适合电商平台主图点击转化的高质量商品图。
```

## 9. `benefit_copy` 功效类产品增强模板

适用于：

- 功效类产品
- 允许少量图中文字辅助转化

```text
请基于参考图生成新的电商商品主图，保持产品主体完全一致，并允许加入少量高转化文案辅助，但文案必须克制、清晰、合规。

产品锁定要求：
- 产品外观、颜色、材质、包装、品牌信息、结构细节必须与参考图一致。
- 产品必须占画面 1/2 以上，成为绝对视觉主体。
- 不允许改变产品，不允许生成相似替代品。

画面目标：
- 输出 1:1 的高质量电商主图。
- 整体应为自然完整的商品图，可以是主图、佩戴图、细节图或场景图，但不要出现明显拼贴、多宫格或低质模板感。
- 视觉上要突出主卖品，提升点击欲望。

文案要求：
- 只允许出现一句大字主文案。
- 文案内容优先表达用户痛点或主卖点。
- 文案必须简短有力，清晰易读。
- 不允许添加多段说明，不允许加无关 slogan。

营销表达要求：
- 可以加入少量符合当地电商习惯的营销术语，例如：{营销表达}
- 必须自然，不要硬翻译，不要过度夸张。

数字表达要求：
- 能数字化表达的卖点优先用数字，例如：{数字信息}
- 数字必须具体明确，不要模糊表达。

配色要求：
- 产品主色、背景色、图标颜色、字体颜色必须统一协调。
- 字体和元素配色必须服务于产品，不要喧宾夺主。

风格要求：
- {风格要求}

禁止项：
- 不要改变产品
- 不要新增无关文字
- 不要新增 logo、水印、平台元素
- 不要添加未经验证的功效承诺
- 不要添加医疗化、夸张化表达
- 不要让背景或文案压过产品主体

请输出一张产品主体强、文案克制但有转化力、适合跨境电商主图使用的高质量图片。
```

## 10. `strict_packaging_only` 模板

适用于：

- 产品包装本身已有关键文字
- 不允许额外新增文案
- 只允许保留包装原文

```text
请基于参考图生成新的电商商品主图，保持产品主体完全一致，并且只允许保留产品包装原有文字内容。

产品锁定要求：
- 产品外观、颜色、材质、包装、品牌信息、结构细节必须与参考图一致。
- 不允许改变产品，不允许生成相似替代品。

画面目标：
- 输出 1:1 的高质量电商主图。
- 画面应为自然完整的商品图，但不要出现明显拼贴、多宫格或低质模板感。
- 主体突出，背景优化，商业感强。

规则：
- 只允许保留产品包装本身原有的文字与 logo。
- 不允许在画面其他区域增加任何新文案。
- 不允许加入营销表达。
- 不允许加入数字标签或说明性图标。

风格要求：
- {风格要求}

场景参考：
- {使用图场景摘要}

禁止项：
- 不要新增文字
- 不要新增 logo
- 不要新增水印
- 不要新增平台元素
- 不要改变产品
- 不要让背景压过主体

请输出一张产品主体突出、画面精致、适合电商主图使用的高质量图片。
```

## 11. 多图生成版

如果需要一次生成多张不同风格，可以在主模板后追加：

```text
请在保持产品主体一致的前提下，输出 {N} 张不同风格的版本。
每张图都要保持 1:1 比例，且风格、背景、摆放方式、场景氛围、光影细节有所区别。
差异可以体现在：
- 背景颜色
- 道具组合
- 拍摄角度
- 主体摆放方式
- 近景/中景构图
- 光线氛围
但所有图片都必须保持统一的商业质感，并持续突出产品主体。
```

## 12. 工作流变量建议

后续接入 OC prompt builder 时，建议只让模型填这些变量：

- `{产品卖点}`
- `{风格要求}`
- `{使用图场景摘要}`
- `{营销表达}`
- `{数字信息}`
- `{已有主图风格摘要}`
- `{目标张数}`
- `{重做范围}`
- `{局部重做说明}`

固定不建议让模型乱改的部分：

- 产品主体锁定要求
- 电商主图要求
- 1:1 比例要求
- 禁止项
- 无文字/无水印/无额外 logo 约束
- 功效类产品的文案数量上限
- 产品主体占比要求

## 13. 重做版模板

适用于：审核打回后局部重做。

```text
请基于上一轮结果继续优化，保持产品主体完全不变，仅针对以下问题进行修改：

本轮仅需修改：
- {重做范围}

问题说明：
- {打回原因}

必须保留：
- 产品本身
- 1:1 比例
- 电商主图质感
- 包装原有内容

不要修改未被指出的问题，不要整体推翻重做，不要改变产品，不要新增文字、logo、水印。
请只围绕本轮问题做精准修正。
```

## 14. 最终建议

第一版工作流里，最稳的方式不是让运营每次自由写提示词，而是：

- 主图数量和副图数量自动判定 `任务模式`
- 不单独开放 `指定副图` 模式
- 一个任务内完成整套图，但主图与副图分阶段执行
- `整套图` 先出主图，主图通过后再做副图
- 副图先做职责规划，再做逐张生成
- 固定三套模板：`visual_only`、`benefit_copy`、`strict_packaging_only`
- OC 只负责判断 `task_mode`、`prompt_mode` 和填充变量
- 人工可以覆盖 `prompt_mode` 判断结果
- 运营只填写卖点、风格要求、补充说明
- 功效类产品再额外填写营销表达和数字信息
- `仅副图` 模式必须提供已有主图或风格参考
- 重做时走单独的重做模板

这样提示词更稳定，图像质量也更容易收敛。
