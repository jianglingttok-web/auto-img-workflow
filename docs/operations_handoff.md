# 运营交接说明

## 当前闭环

当前本地版本已经支持以下闭环：

1. 运营在飞书表提交任务。
2. 本地 worker 自动拉取任务并生成主图。
3. 预览图直接发送到提交人/审核人飞书私聊。
4. 审核人直接在私聊回复通过或自然语言打回意见。
5. 系统先回文本进度消息，再自动继续下一轮生成或交付。
6. 最终结果和压缩包直接发送到飞书私聊。

## 本地启动

首次或重启前建议先确认 `.env` 已配置：

- `FEISHU_APP_ID`
- `FEISHU_APP_SECRET`
- `ARK_API_KEY`
- `ARK_TEXT_MODEL`
- `SEEDREAM_MODEL`

启动：

```bat
scripts\start_ops_runtime.bat
```

停止：

```bat
scripts\stop_ops_runtime.bat
```

查看运行状态：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\status_ops_runtime.ps1
```

## 审核消息规范

支持两类回复：

- 通过：`通过 cck04`
- 打回：`打回 cck04 审核意见：主图比例太大，请参考副图1尺寸`

如果当前只有一个待审核任务，也可以直接回复自然语言意见，例如：

- `主图佩戴角度太正，想更偏侧脸和锁骨特写`

系统会把自然语言意见交给豆包整理成新的制作指令后再发起重做。

## 正常反馈顺序

当审核人发出意见后，正常会依次看到：

1. `已收到 ... 正在解析并准备重做。`
2. `已启动 ... 第N轮主图/副图重做，完成后会自动发送预览。`
3. 新一轮预览图。

如果是通过，则会看到：

1. `已收到 ... 通过意见，正在处理后续流程。`
2. `已启动副图生成` 或 `正在整理并发送交付资料。`
3. 副图预览或最终交付。

## 常见异常

### 1. 发了消息但没有任何反馈

先检查进程：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\status_ops_runtime.ps1
```

如果没有运行中的 `run-feishu-long-connection` 或 `run-feishu-image-worker`，重新执行：

```bat
scripts\start_ops_runtime.bat
```

### 2. 回复后提示找不到任务

说明当前消息没有匹配到待审核任务。优先确认：

- 任务仍处于待审核状态。
- 消息里是否带了正确的任务 ID。
- 同一审核人是否同时挂了多个待审核任务。

### 3. 图片迟迟不出

先看飞书是否已经收到“已启动重做/生成”的文本回执。

- 如果收到了：说明审核消息已经进入本地流程，继续查模型或生图耗时。
- 如果没收到：优先查长连接进程是否在线。

## 当前边界

当前本地版本适合先投入运营验证，但仍然是本地常驻：

- 机器必须开机。
- 网络必须稳定。
- 仍建议后续整体迁云。

迁云时优先迁移：

1. 长连接接收器
2. 本地 worker
3. 再迁运行目录和日志目录
